GAIA is a proactive personal AI assistant built as a full-stack Nx monorepo. This documentation covers how to set up, configure, and deploy GAIA in production.

## What's in this repo

| App / service | Language | Purpose |
|---|---|---|
| `apps/web` | Next.js | Web frontend |
| `apps/desktop` | Electron + Next.js | Desktop app |
| `apps/mobile` | React Native (Expo) | Mobile app |
| `apps/api` | FastAPI + LangGraph | Backend API |
| `apps/bots/discord` | TypeScript | Discord bot |
| `apps/bots/slack` | TypeScript | Slack bot |
| `apps/bots/telegram` | TypeScript | Telegram bot |
| `apps/voice-agent` | Python | LiveKit voice worker |

Shared code lives in `libs/shared/py` (Python) and `libs/shared/ts` (TypeScript).

## Infrastructure

The production stack runs on a GCP VM via Docker Compose. Observability (Prometheus, Loki, Grafana) is managed from the separate [logging-grafana](https://github.com/theexperiencecompany/logging-grafana) repo.
