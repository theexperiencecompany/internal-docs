All three bots share a single `apps/bots/.env` file. In production, they run as Docker containers with the `bots` profile.

```bash
docker compose -f docker-compose.prod.yml --profile bots up -d
```

---

## Discord

### Developer Portal setup

<Steps>
  <Step title="Create an application">
    Go to [discord.com/developers/applications](https://discord.com/developers/applications) and create a new application.
  </Step>
  <Step title="Create a bot">
    In the application settings, go to **Bot** and click **Add Bot**.

    Under **Privileged Gateway Intents**, enable:
    - **Server Members Intent**
    - **Message Content Intent**
  </Step>
  <Step title="Get the bot token">
    In the **Bot** section, click **Reset Token** and copy the token. Set it as `DISCORD_BOT_TOKEN` in `apps/bots/.env`.
  </Step>
  <Step title="Get the client ID">
    In **OAuth2 → General**, copy the **Client ID**. Set it as `DISCORD_CLIENT_ID` in `apps/bots/.env`.

    Also set it as `DISCORD_OAUTH_CLIENT_ID` in `apps/api/.env` if you want one-click account linking.
  </Step>
  <Step title="Register slash commands">
    Slash commands must be registered with Discord before they appear. Run:

    ```bash
    cd apps/bots
    pnpm deploy-commands
    ```

    This is a one-time operation (or run again when commands change). In production, it runs automatically as part of the CI/CD pipeline on deploys that affect the Discord bot.
  </Step>
  <Step title="Invite the bot to a server">
    In **OAuth2 → URL Generator**, select scopes: `bot`, `applications.commands`.

    Required bot permissions:
    - Send Messages
    - Read Message History
    - Add Reactions
    - Use Slash Commands

    Use the generated URL to invite the bot.
  </Step>
</Steps>

### Required env vars

```bash
# apps/bots/.env
DISCORD_BOT_TOKEN=<bot-token>
DISCORD_CLIENT_ID=<client-id>

# apps/api/.env
DISCORD_BOT_TOKEN=<same-bot-token>       # for notification adapter
DISCORD_OAUTH_CLIENT_ID=<client-id>      # for one-click account linking
DISCORD_OAUTH_CLIENT_SECRET=<secret>     # for one-click account linking
```

## Slack

The Slack bot uses Socket Mode (WebSocket-based). No public webhook URL is required.

### App setup

<Steps>
  <Step title="Create a Slack app">
    Go to [api.slack.com/apps](https://api.slack.com/apps) and click **Create New App → From Scratch**.
  </Step>
  <Step title="Enable Socket Mode">
    In **Socket Mode**, toggle it on. Create an **App-Level Token** with the `connections:write` scope. Set it as `SLACK_APP_TOKEN`.
  </Step>
  <Step title="Set OAuth scopes">
    In **OAuth & Permissions → Bot Token Scopes**, add:
    - `app_mentions:read`
    - `channels:history`
    - `chat:write`
    - `commands`
    - `groups:history`
    - `im:history`
    - `im:read`
    - `im:write`
    - `mpim:history`
    - `users:read`
  </Step>
  <Step title="Enable events">
    In **Event Subscriptions**, enable events and subscribe to:
    - `app_mention`
    - `message.im`
    - `message.channels`
    - `message.groups`
    - `message.mpim`
  </Step>
  <Step title="Install the app">
    In **OAuth & Permissions**, click **Install to Workspace**. Copy the **Bot User OAuth Token** and set it as `SLACK_BOT_TOKEN`.
  </Step>
  <Step title="Copy the signing secret">
    In **Basic Information**, copy the **Signing Secret** and set it as `SLACK_SIGNING_SECRET`.
  </Step>
</Steps>

### Required env vars

```bash
# apps/bots/.env
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
SLACK_SIGNING_SECRET=...

# apps/api/.env (for one-click account linking)
SLACK_OAUTH_CLIENT_ID=...
SLACK_OAUTH_CLIENT_SECRET=...
```

## Telegram

### BotFather setup

<Steps>
  <Step title="Create a bot">
    Open Telegram and search for **@BotFather**. Send `/newbot` and follow the prompts.

    Set the bot token as `TELEGRAM_BOT_TOKEN` in `apps/bots/.env` and `apps/api/.env`.
  </Step>
  <Step title="Set the username (optional)">
    Set `TELEGRAM_BOT_USERNAME` in `apps/api/.env` if you want the API to reference it in messages (e.g. `heygaia_bot`).
  </Step>
  <Step title="Configure commands (optional)">
    Send `/setcommands` to @BotFather to register visible commands in the Telegram UI.
  </Step>
</Steps>

### Required env vars

```bash
# apps/bots/.env
TELEGRAM_BOT_TOKEN=...

# apps/api/.env
TELEGRAM_BOT_TOKEN=...         # for notification adapter
TELEGRAM_BOT_USERNAME=...      # optional
```

## Running bots locally

Each bot can be run individually:

```bash
cd apps/bots
pnpm discord    # Discord bot
pnpm slack      # Slack bot
pnpm telegram   # Telegram bot
```

Make sure `GAIA_API_URL` points to a running API instance and `GAIA_BOT_API_KEY` matches the API's key.

---

## Shared requirements

All bots depend on the API being healthy. In Docker Compose, they wait for `gaia-backend` to pass its health check before starting.

The `GAIA_BOT_API_KEY` must match exactly between `apps/bots/.env` and `apps/api/.env`. If they differ, the bots cannot authenticate with the API.

Inside Docker Compose, bots reach the API via the service name:

```bash
GAIA_API_URL=http://gaia-backend
```

For local development (outside Docker), use:

```bash
GAIA_API_URL=http://localhost:8000
```
